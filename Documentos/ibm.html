1. Introducción

1.1 Propósito

El propósito de este documento es proporcionar una descripción completa de los requisitos para el desarrollo del "Sistema de Gestión Integral Zona Web Lara". Este sistema tiene como objetivo automatizar y centralizar el control de las operaciones diarias, supliendo las carencias actuales del negocio mediante tecnología de la información.

1.2 Alcance

El software a desarrollarse abarcará la recepción y atención de clientes, el bloqueo y desbloqueo de equipos por tiempo de uso, la generación de alertas, el manejo de punto de venta (POS), el seguimiento de inventario físico y digital, la impresión de formatos y la gestión de flujos de trámites legales. No incluye módulos de contabilidad financiera integral ni nómina, limitándose al control operativo.

2. Descripción General

2.1 Perspectiva del Producto

El sistema es autónomo y no depende de otros sistemas externos más que del hardware periférico (impresoras y red LAN) para el bloqueo de PCs. Utilizará una base de datos local relacional para garantizar la persistencia de la información ante cortes de energía o fallos de red.

2.2 Funcionalidades del Producto

  • Módulo Cyber Control: Monitoreo de disponibilidad, asignación de equipos, cronometraje, bloqueo remoto automático, alertas GUI iterativas y extensión de tiempo.
  • Módulo Punto de Venta (POS): Venta de productos varios, generación de factura y descuento automático de stock.
  • Módulo Inventario y Proveedores: Gestión de cantidades (agregar/eliminar/editar), monitoreo contra cantidad mínima, alertas a dueña, y registro de proveedores frecuentes.
  • Módulo Impresión: Recepción de formatos, llenado digital y enrutamiento a impresión (B/N o Color).
  • Módulo Asesoría Legal: Triage de documentos (controlados vs no controlados), redirección de flujos y registro de tramites.

3. Requisitos Específicos

3.1 Interfaces de Usuario

El sistema presentará un Dashboard principal post-Login. Para el Cyber Control, se usarán colores semánticos (Verde Disponible, Rojo Bloqueado). Las alertas por falta de atención serán ventanas modales con opción a "Repetir Alerta".

3.2 Interfaces de Hardware

  • Servidor/Host: PC con mínimo 8GB RAM y procesador i3 (soporte para BD y conexión simultánea de hasta 20 PCs).
  • Red: Switch o Router con red LAN (cableada preferiblemente) para envío de paquetes de bloqueo a las PCs clientes.
  • Periféricos: Impresoras multifunción conectadas por red al servidor.

3.3 Interfaces de Software

  • Sistema Operativo Base: Windows 10/11.
  • Motor de Base de Datos: MySQL 8.0 o PostgreSQL.
  • Runtime: Entorno de ejecución según el lenguaje elegido (Ej: JRE para Java, .NET Core para C#).

5. Requerimientos No Funcionales

ID Atributo de Calidad Descripción del Requerimiento Prioridad
RNF-01 Seguridad Gestión de usuarios robusta. Las credenciales deben estar encriptadas en la base de datos (Hashing). Alta
RNF-02 Fiabilidad El sistema debe tener un mecanismo de "Fallo Seguro": si se cae el servidor, las PCs cliente deben bloquearse automáticamente por seguridad. Alta
RNF-03 Rendimiento El tiempo de respuesta entre que el cronómetro llega a cero y la PC del cliente se bloquea no debe exceder los 2 segundos. Alta
RNF-04 Usabilidad El diseño debe evitar la redundancia. El flujo de Cyber Control no debe requerir más de 3 clics para iniciar una sesión. Media
RNF-05 Mantenibilidad El código debe estar segregado por módulos (MVC) para permitir modificar el módulo legal sin afectar el módulo de inventario. Media
